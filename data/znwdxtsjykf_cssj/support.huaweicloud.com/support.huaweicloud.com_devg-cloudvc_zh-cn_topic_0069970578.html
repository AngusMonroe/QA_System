查询会议信息

管理员或EC帐号可以通过该接口查询指定会议详情。管理员可以查询管理权限域内所有的会议，EC帐号仅能查询当前帐号管理的会议。

注意事项

暂时不支持子会议查询。

接口说明

接口访问路径：

http://{ip}:{port}/conferences/{conferenceid}
请求方法： 
GET

参数格式：

application/json;charset=UTF-8

如果是非周期性会议，该字段不携带或为“0”；如果是周期性会议，该字段为子会议的ID。

startTime

endTime

String

会议结束时间 。格式：YYYY-MM-DD HH:MM。

mediaTypes

String

会议的媒体类型。

  • Voice：语音
  • Video：标清视频
  • HDVideo：高清视频（与Video互斥，如果同时选择Video、HDVideo，则系统默认选择Video）
  • Telepresence：智真视频（与Video、HDVideo互斥）
  • Data：多媒体
  • DesktopSharing：桌面共享（只能与Voice配合使用，且与Data互斥，表示由BFCP辅流提供的桌面共享）

conferenceState

String

会议状态。

  • Schedule：预定状态
  • Creating：正在创建状态
  • Created：会议已经被创建，并正在召开
  • Destroyed：会议已经关闭

accessNumber

String

会议接入码。

passwordEntry

PasswordEntry

会议密码条目。

预订者查询时返回主席密码和来宾密码，主席查询时返回主席密码，来宾查询时返回来宾密码。

accountID

String

会议预订者帐号。

scheduserName

String

会议预订者帐号名称。

confVideoMode

String

会议视频模式。

  • Free：自由选看模式。仅适用于智真会议。
  • Fixed：固定画面广播模式。固定广播使用该枚举值。
  • AutoBrowse：自动浏览，也叫轮询模式。轮询广播使用该枚举值。
  • VAS：声控切换模式。启用声控使用该枚举值。
  • RollCall：点名模式。仅查询在线会议时有效。

multiStreamFlag

int

是否为多流视频会议。

  • 0：单流会议
  • 1：多流会议

cycleParams

CycleParams

周期性会议参数。当会议是周期性会议时携带该参数。

该参数包括周期性会议的开始日期、结束日期、会议的周期和周期中的开会时间点。

Int
  • 周期类型选择“Day”，interval取值同point，可忽略，表示每几天召开一次。
  • 周期类型选择“Week”，interval取值表示隔几周。如填2，则表示每两周的point召开。
  • 周期类型选择“Month”，interval取值表示隔几月。如填2，则表示每两月的point召开。

默认值都为1。

表7 周期性会议召开点参数 WeekDayParam

point

Int[0..*]

周期内的会议召开点。例如：

  • 周期类型选择“Day”，point为2，则表示每两天召开一次。
  • 周期类型选择“Week”，point中为1和3，则表示每周一和周三召开会议。0表示周日。
  • 周期类型选择“Month”，point中为12和20，则表示每个月的12号和20号召开会议。
