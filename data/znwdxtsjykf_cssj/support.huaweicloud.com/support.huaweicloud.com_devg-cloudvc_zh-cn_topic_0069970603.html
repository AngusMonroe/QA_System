增加企业管理员

该接口提供增加企业管理员的功能，服务器收到请求，增加企业管理员并返回结果。

注意事项

  • 当adminType为COMMON_ADMIN（普通管理员）时，该接口将原有的普通成员提升为普通管理员。此时pwd字段留空，管理员的密码为该成员原有的密码。
  • 当adminType为DEFAULT_ADMIN时，pwd必填。

接口说明

接口访问路径：

http://{ip}:{port}/corp/{corpId}/userAccount
HTTP 协议请求方式： 
POST

参数格式：

application/json;charset=UTF-8

参数描述

表1 输入参数描述

参数名

数据类型

约束

说明

Request URI

corpId

String

1

企业ID，取值长度不超过31。

Request Header

Authorization

String

1

接口鉴权头域。包括HTTP基本认证标识（Basic）和Base64编码的token字符串。

Request Body

userAccount

String

1

用户帐号

  1. 长度不超过128位
  2. 字符集0时: 不支持中文、非全角字符、空格和< > ( ) \ / & [ ] , ; : “ ? * # ‘ ! % ^ = | 等特殊字符
  3. 字符集1或者2时: 不支持中文、空格和\ / : * ? ” < > | 九个特殊字符

userType

String

1

用户类型。

  • CORP_MEMBER（企业成员账户）
  • UPATH（upath账号）
  • IPPHONE（ip话机账号）
  • TE（TE终端账号）
  • CUSTOMER（顾客账号）
  • PUBLIC_DEVICE（公共设备账号）

adminType

String

0..1

userType 选CORP_MEMBER时必填。

管理员类型：

  • DEFAULT_ADMIN（默认（超级）管理员）
  • COMMON_ADMIN（普通管理员）
  • NOT_ADMIN（非管理员）
说明：

默认（超级）管理员仅可以是admin

isActive

Int

1

是否激活

  • 0：未激活
  • 1：激活

emailAddr

String

0..1

电子邮箱地址

pwd

String

0..1

登录密码，取值要求如下：

  1. 长度最小8位
  2. 密码不能和账号相同
  3. 密码不能和账号的反转字符串相同
  4. 密码不能是账号的部分
  5. 密码至少要包含数字、大写字母、小写字母中的2种
说明：
  • 当adminType为COMMON_ADMIN（普通管理员）时，该参数需要留空。
  • 当adminType为DEFAULT_ADMIN时，该参数必填。

roleName

String

1

管理员角色名称

参数格式为：企业管理员菜单权限模板的值+"@"+企业账号

userIndividual

UserIndividual

1

企业成员的个人信息

userContactInfo

0..1

企业成员的联系信息

numSegList

personRoleName

0..1
