二次开发流程

  1. 二次开发总体流程

    基于企业云通信平台进行二次开发总体上由以下五个骤组成：

    1. 注册华为云

      登录华为云：http://www.huaweicloud.com/后，单击“注册”，按照界面提示完成华为云的注册。成功注册后请尽快完成实名认证。

    2. 订购华为云会议服务

      登录华为云：http://www.huaweicloud.com/后，在首页选择“产品 > 企业应用 > 会议”，进入会议首页。

      在首页查看会议介绍，需要订购会议服务请拨打售前咨询电话：4000-955-988 转1。

    3. 下载二次开发资源

      二次开发指南：面向开发者提供二次开发端到端指导，通过快速入门和典型应用开发场景，循序渐进地指导开发人员完成基于平台/产品提供的开放API或SDK进行二次开发，以最终实现某个特性或功能。

      接口参考：包含接口分类说明、方法定义、参数描述、返回值、错误码等，通过多种方式对API信息进行组织，根据平台的形态，提供较好的信息索引方式，便于开发者查询具体接口的详细信息。

      SDK开发包：将企业云通信的实时音视频、数据协作等电信能力按不同的平台进行封装，通过API接口的形式对外开放，支持Windows、Android、iOS等平台。第三方应用开发商/开发者获得相应的许可后，可以对开放能力进行调用，丰富现有应用的业务体验。

      样例代码：提供统一鉴权与登录、音视频呼叫、音视频会议、桌面协同与共享等功能的全量示例代码，开发人员通过看示例代码可快速掌握相关功能的集成方法。

    4. 集成开发和测试

      二次开发主要涉及服务器REST集成和客户端SDK集成，根据需求场景选择集成方式。

      开发者准备好相应平台的开发环境，参考开发指南和示例代码，根据业务场景在应用中集成需要的通信能力。此阶段可申请连接远程实验室接入进行应用调试和功能测试。

    5. 商用发布

      开发者完成应用开发和调测后可以接入到企业云通信的生成环境，并将应用发布给最终用户使用。

  2. 服务器REST集成

    1. 服务器地址

      订购SaaS业务后，可获取企业云通信 REST API的服务接入地址。

      订购前可免费连接华为远程实验室环境，参考远程实验室接入

    2. 传输安全

      支持通过HTTP或HTTPS通道进行请求通信。为了获得更高的安全性，推荐您使用HTTPS通道发送请求。

    3. HTTP编码

      请求及返回结果都使用UTF-8字符集进行编码。

    4. 帐号模型

      企业云通信支持SP、企业、员工多级管理模型；开发者订购企业云通信业务后就拥有了企业的创建和管理权限，可通过Console或业务发放接口创建企业和员工账号(EC账号)，例如创建帐号：w00255686@huawei.com 密码：1qaz@WSX

      开发者的第三方应用集成企业云通信的能力时需要为第三方应用的用户帐号绑定EC帐号，第三方应用的用户账号和EC帐号是一对一映射的。

    5. 登录鉴权

      第三方WEB服务器使用业务接口之前需要先调登录鉴权接口，并成功获取认证Token。企业云通信采用HTTP Diget鉴权机制，通过MD5算法对用户名和密码进行加密。登录鉴权Token获取步骤如下图所示：

    • 发送登录请求
      开发者发送一个登录请求给服务器，具体格式如下：

      HTTP请求格式：

      POST https://{APIGW_IP}:{PORT}/login HTTP/1.1

      Authorization: Digest username={username}, algorithm=MD5

      HTTP请求样例：

      POST https://10.174.0.120:8543/login HTTP/1.1

      Authorization: Digest username=enter1@huawei.com, algorithm=MD5
    • 获取401挑战响应

      发送登录请求后，开发者会获取一个401挑战响应。

      401挑战响应样例：

      HTTP/1.1 401 Unauthorized

      WWW-Authenticate: Digest auth-name="enter1@huawei.com", nonce="150172725883013857027497582144236969991632672", realm="huawei.com"

      m-trace: [22/Jul/2017:09:26:00.970] https-in 0AA6886B_D0FD_0AAE0078_2184_59731A28_0A89_1D27

      Content-Type: text/html;charset=UTF-8

      Accept-Ranges: bytes

      Date: Sat, 22 Jul 2017 09:26:01 GMT

      Transfer-Encoding: chunked

      Connection: close

      收到响应后，首先获取响应头WWW-Authenticate中的realm和nonce，然后按照以下方法计算response的值，其中MD5采用32位小写加密算法。

      response计算方法：

      HA1=MD5(username:realm:password)

      HA2=MD5(POST:/login/sc)

      response=MD5(HA1:nonce:HA2)

      response计算样例：

      HA1=MD5(enter1@huawei.com:huawei.com:1qaz@WSX)

      HA2=MD5(POST:/login/sc)

      response=MD5(HA1:150172725883013857027497582144236969991632672:HA2)
    • 发送鉴权认证

      开发者根据401挑战响应计算出response值后，发送鉴权认证请求给服务器，具体格式如下：

      HTTP请求格式：

      POST https://{APIGW_IP}:{PORT}/login HTTP/1.1

      Authorization: Digest username={username}, c={realm}, nonce={nonce}, uri="/login/sc", response={response}

      Host: {APIGW_IP}:{PORT}

      HTTP请求样例：

      POST https://10.174.0.120:8543/login HTTP/1.1

      Authorization: Digest username="admin", c="huawei.com", nonce="150172158330044896625513661666677719826696698", uri="/login/sc", response="49eb4676b8a78a913a6f30e22f4b679b"

      Host: 10.174.0.120:8543
    • 获取鉴权成功响应

      开发者发送鉴权认证请求后会获取鉴权成功的响应，解析消息后获取<AccessToken>标签内的值即为token。

    鉴权成功响应样例：

    HTTP/1.1 200 OK

    m-trace: [22/Jul/2017:09:29:16.597] https-in 0AA6886B_D14C_0AAE0078_2184_59731AEC_0AA0_1D27

    Content-Type: text/xml;charset=UTF-8

    Accept-Ranges: bytes

    Date: Sat, 22 Jul 2017 09:29:16 GMT

    Transfer-Encoding: chunked

    Connection: close

    <?xml version="1.0" encoding="UTF-8"?>

    <UserInfo>

    <LoginToken>

    <AccessToken>MTUwMTcyNzI2MzQ4MTE5ODM4MDkxMzQ2MjE5Nzg4NDM1NDIwNjMzMDc5MzAzOjExMDBAMTAuMTc0LjE0LjU5</AccessToken>

    <TokenType>DOMAIN</TokenType>

    <ExpireIn>63815</ExpireIn>

    </LoginToken>

    <FirstLogin>NO</FirstLogin>

    <PWDExpireTime>89</PWDExpireTime>

    <RealUserAccount>enter1@huawei.com</RealUserAccount>

    </UserInfo>
  3. 客户端SDK集成
    1. CloudEC SDK是什么？

      CloudEC SDK，是一个基于企业云通信业务场景的客户端LIB 库，集成了对媒体（音视频转发和转码、丢包补偿、回音消除等）和信令(注册、路由、呼叫控制等)的处理能力,对外提供简单的API，开发者不需要关注底层的技术实现细节。第三方应用开发商/开发者获得相应的许可后，可以对相关开放能力进行调用，丰富和增强您现有应用的业务体验。

      CloudEC SDK支持Android、iOS、Windows平台，配套提供了开发指南、接口参考和客户端Demo。其中客户端Demo提供一系列Sample Codes演示如何调用接口，帮助您完成集成企业云通信实时音视频能力的相关业务开发。

    2. CloudEC SDK架构

      架构分层

      功能描述

      场景化适配层

      基于业务层的原子C接口，进行场景化封装，提供High-Level API 支持C、Java和JavaScript语言。企业云通信的标准客户端、客户端Demo和第三方客户端应用使用的是同一套开放接口。

      业务能力层

      提供音视频呼叫、音视频会议、数据协作、即时消息等企业通信能力，开发者可根据所需的业务场景选择加载相应的业务模块。

      公共组件层

      提供协议、媒体、安全等基础组件能力；支持SIP、SDP、H323、FTP、HTTP、LDAP、802.1X、DNS、DHCP、SSH等协议；支持G.711a/G.711μ/G.729AB/G.722/ iLBC/Opus等多种语音编解码， 支持H264、H263等多种视频编解码。

      OS适配层

      针对不同操作系统平台，提供定时器、时间、内存检查、 锁、信号量、共享内存、CPU、文件、socket、线程等机制的统一适配，实现跨平台能力。
    3. CloudEC SDK的使用

      第三方客户端应用，通过集成CloudEC SDK对接企业云通信，如何获取SDK请参见资源下载。开发者快速上手，可通过参考开发指南中的HelloWorld，导入SDK，完成相应环境的准备。进一步深入了解某个能力如何集成，需要学习开发指南和接口参考的具体章节，另外Demo中的代码样例是您二次开发过程中的得力助手。

      CloudEC SDK是基于企业的员工账号（EC帐号）鉴权，使用SDK能力前先要完成鉴权登录。开发者订购企业云通信业务后就拥有了企业的创建和管理权限，可通过Console或业务发放接口创建企业和员工账号，例如创建帐号：w00255686@huawei.com 密码：1qaz@WSX。
