创建集群并执行作业

功能介绍

创建一个MRS集群，并在集群中提交一个作业。该接口不兼容Sahara。

支持同一时间并发创建10个集群。

使用接口前，您需要先获取如表1所示的资源信息。

表1 获取资源

资源

VPC

请参见虚拟私有云《接口参考》中“VPC > 查询VPC列表”和“VPC > 创建VPC”的操作指导。

子网

请参见虚拟私有云《接口参考》中“子网 > 查询子网列表”和“子网 > 创建子网”的操作指导。

区域

区域和可用分区信息请参见终端节点及区域说明。

MRS目前支持MRS 1.3.0和MRS 1.5.0版本。默认值为当前最新版本，当前的最新版本为MRS 1.5.0。

    cluster_version

    否

    String

    集群版本。

    MRS目前支持MRS 1.3.0和MRS 1.5.0版本。默认值为当前最新版本，当前的最新版本为MRS 1.5.0。

    cluster_type

    否

    Integer

    集群类型。

    • 0：分析集群
    • 1：流式集群

    默认值为0：分析集群。

    volume_type

    是

    String

    磁盘存储类别，目前支持SATA、SAS和SSD。

    • SATA：普通I/O
    • SAS：高速I/O
    • SSD：超速I/O

    volume_size

    是

    Integer

    Core节点数据磁盘存储空间。为增大数据存储容量，创建集群时可同时购买磁盘。有如下应用场景：

    • 数据存储和计算分离，数据存储在OBS系统中，集群费用相对较低，计算性能不高，并且集群随时可以删除，建议数据计算不频繁场景下使用。
    • 数据存储和计算不分离，数据存储在HDFS中，集群费用相对较高，计算性能高，集群需要长期存在，建议数据计算频繁场景下使用。

    取值范围：100GB～1000GB

    node_public_cert_name

    是

    String

    密钥对名称。用户可以使用密钥方式登录集群Master节点。

    safe_mode

    是

    Integer

    MRS集群运行模式。

    • 0：普通集群，表示Kerberos认证关闭，用户可使用集群提供的所有功能。
    • 1：安全集群，表示Kerberos认证开启，普通用户无权限使用MRS集群的“文件管理”和“作业管理”功能，并且无法查看Hadoop、Spark的作业记录以及集群资源使用情况。如果需要使用集群更多功能，需要找MRS Manager的管理员分配权限。

      仅当“safe_mode”配置为“1”时，请求消息体中包含cluster_admin_secret字段。

    cluster_admin_secret

    否

    String

    配置MRS Manager管理员用户的密码。

    MRS 1.5.0需要满足：

    • 密码长度应在6～32个字符之间
    • 必须包含如下5种中至少2种字符的组合
      • 至少一个小写字母
      • 至少一个大写字母
      • 至少一个数字
      • 至少一个特殊字符：`~!@#$%^&*()-_=+\|[{}];:'",<.>/?
      • 空格
    • 不能和用户名一样
    • 不能和用户名的倒写一样

    MRS 1.3.0需要满足：

    • 密码长度应在8～64个字符之间
    • 必须包含如下5种中至少4种字符的组合
      • 至少一个小写字母
      • 至少一个大写字母
      • 至少一个数字
      • 至少一个特殊字符：`~!@#$%^&*()-_=+\|[{}];:',<.>/?
      • 空格
    • 不能和用户名一样
    • 不能和用户名的倒写一样

    仅当“safe_mode”配置为“1”时需要配置此参数。

    log_collection

    否

    Integer

    集群安装失败时，是否搜集日志。

    • 0：不收集
    • 1：收集

    默认设置为0。

    当log_collection设置为1时，OBS桶将被创建用于MRS服务系统日志收集，可能产生一定的费用。

    job_type

    是

    Integer

    作业类型码。

    • 2：Spark

    jar_path

    是

    String

    执行程序Jar包地址，需要满足如下要求：

    • 最多为255字符，不能包含;|&>,<'$特殊字符，且不可为空或全空格。
    • 需要以“/”或“s3a://”开头。
    • Spark Script需要以“.sql”结尾，MapReduce和Spark Jar需要以“.jar”结尾，sql和jar不区分大小写。

    arguments

    否

    String

    程序执行的关键参数，该参数由用户程序内的函数指定，MRS只负责参数的传入。

    最多为255字符，不能包含;|&>'<$特殊字符，可为空。

    input

    否

    String

    数据输入地址，必须以“/”或“s3a://”开头。请配置为正确的对象存储服务（OBS）

    最多为255字符，不能包含;|&>'<$特殊字符，可为空。

    output

    否

    String

    数据输出地址，必须以“/”或“s3a://”开头。请配置为正确的OBS路径，如果该路径不存在，系统会自动创建。

    最多为255字符，不能包含;|&>'<$特殊字符，可为空。

    job_log

    否

    String

    作业日志存储地址，该日志信息记录作业运行状态。必须以“/”或“s3a://”开头，请配置为正确的OBS路径。

    最多为255字符，不能包含;|&>'<$特殊字符，可为空。

    shutdown_cluster

    否

    Bool

    作业执行完成后，是否删除集群。

    • true：是
    • false：否

    file_action

    否

    String

    数据导入导出。

    • import
    • export

    submit_job_once_cluster_run

    是

    Bool
    • true：创建集群同时提交作业
    • false：单独提交作业

    此处应设置为true。

    hql

    否

    String

    HQL脚本语句。

    hive_script_path

    是

    String

    sql程序路径，仅Spark Script和Hive Script作业需要使用此参数。需要满足如下要求：

    • 最多为255字符，不能包含;|&><'$特殊字符，且不可为空或全空格。
    • 需要以“/”或“s3a://”开头。
    • 需要以“.sql”结尾，sql不区分大小写。
