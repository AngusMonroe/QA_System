修改预约会议

管理员可以通过该接口修改预约会议。

注意事项

Cloud VC解决方案不支持修改周期性会议的单个子会议。

接口说明

接口访问路径：

http://{ip}:{port}/conferences/{conferenceid}
请求方法： 
PUT

参数格式：

application/json;charset=UTF-8

参数描述

表1 输入参数描述

参数名

类型

约束

描述

Request URI

conferenceid

String

1

会议标识。

Request Header

Authorization

String

1

授权令牌。

Request Body

size

Int

1

会议方数，取值范围为3~100。取值范围由各AS确定。

timeZoneID

String

0..1

国际时区ID描述字符，取值参见国际时区。

例如："GMT-12:00"

startTime

String

0..1

会议开始时间，使用UTC时间。预定创建会议时，如果没有指定开始时间，或填入空串，则表示会议马上开始。

预约周期会议时，该参数必填。

格式：YYYY-MM-DD HH:MM。

说明：

当传入错误时间时，服务器会将时间字符串转换为绝对时间。例如2017年13月转换后为2018年1月。

建议通过客户端校验或时间插件来保证日期格式的正确性。

length

Int

0..1

会议持续时长，单位分钟，默认时长为1小时。默认时长由会议AS定义。

subject

String

0..1

会议主题。长度限制默认为128个字符。默认值由会议AS定义，MediaX为128个字符。

mediaTypes

String

1

会议的媒体类型。由1个或多个枚举String组成，多个枚举时，每个枚举值之间通过逗号（,）分隔。支持如下枚举值：

  • Voice：语音
  • Video：标清视频
  • HDVideo：高清视频（与Video互斥，如果同时选择Video、HDVideo，则系统默认选择Video）
  • Data：多媒体
  • DesktopSharing：桌面共享（只能与Voice配合使用，且与Data互斥，表示由BFCP辅流提供的桌面共享）

CloudPBX组网时，Data需要与Voice组合使用。

welcomeVoiceEnable

Int

0..1

入会欢迎词开关，默认值为1。由会议AS定义。取值如下：

  • 0：不提示
  • 1：提示音

enterPrompt

Int

0..1

成员入会提示音开关，默认值由会议AS定义。取值如下：

  • -1：无提示音
  • -2：默认提示音
  • -3：提示音（DU）
  • -4：广播与会者姓名

leavePrompt

Int

0..1

成员离会提示音开关，默认值由会议AS定义。取值如下：

  • -1：无提示音
  • -2：默认提示音
  • -3：提示音（DU）
  • -4：广播与会者姓名

attendees

Attendee[0..*]

0..n

预定会议时，指定的与会者列表。

该与会者列表可以用于发送会议通知、会议提醒、会议开始时进行自动邀请。

reminders

Reminder[0..*]

0..n

指定各种会议提醒的时间点。

不指定表示不设置提醒；设置了提醒，则会议会根据attendees指定的各种类型的地址进行提醒。

cycleParams

CycleParams

0..1

周期性会议的参数。

当会议是周期性会议时，该参数必须填写，否则服务器会忽略该参数。

该参数包括周期性会议的开始日期、结束日期、会议的周期和周期中的开会时间点。

isAllowRecord

Int

0..1

会议是否启用网络录制，默认为0。

  • 1：启用录制
  • 0：不启用录制

当前仅支持语音录制，默认不启用录制。

isAutoRecord

Int

0..1

会议是否自动启动录制。该参数只有在isAllowRecord为1的情况下才生效。

  • 1：自动启动录制
  • 0：不自动启动录制

当前仅支持语音录制，默认不启用录制。

userType

String

0..1

该参数表示预定者的用户类型。

  • WEB：WEB方式
  • MOBILE：移动软终端
  • PC：PC软终端

不携带，则认为是其他类型。

encryptMode

Int

0..1

会议媒体加密模式。枚举值如下：

  • 0：auto——自适应加密
  • 1 ：must——强制加密
  • 2：不出现——不加密

language

String

0..1

会议的默认语言，默认值由会议AS定义。对于系统支持的语言，按照RFC3066规范传递。

  • zh-CN：简体中文
  • en-US：美国英文

autoProlong

Int

0..1

是否自动延长会议。

  • 1 ：自动延长
  • 0：不自动延长

assistantMediaParams

AssistantMediaParams

0..1

辅流参数，该参数对视频会议有效。

multiStreamFlag

Int

0..1

是否为多流视频会议，默认值为0。枚举值如下：

  • 0：单流会议
  • 1：多流会议

name

role

  • singleCiscoTP：思科单屏
  • threeCiscoTP：思科三屏
  • h323：H.323网络用户
表3 会议提醒 Reminder

参数名

类型

约束

描述

type

AddressType

1

地址类型。String类型的枚举值，取值如下：

  • email：电子邮件通知
  • sms：短信通知

time

long

1

会议通知的时间点，填写相对于会议开始时间的时间。单位：分钟。

表4 周期会议参数 CycleParams

参数名

类型

约束

描述

startDate

String

1

周期性会议的开始日期。格式：YYYY-MM-DD。

说明：

建议通过客户端校验或时间插件来保证日期格式的正确性。

endDate

String

0..1

周期性会议的结束日期。格式：YYYY-MM-DD。

说明：

endDate字段与cycleConut字段不能同时为空，两者必须至少填写一项。当两者同时填写时，优先以endDate为准。

cycleCount

Int

0..1

重复次数。内置会议最大为30次。

cycle

String

1

周期类型。

  • Day：天（年作为范围）
  • Week：星期（年作为范围）
  • Month：月（年作为范围）

interval

Int

0..1
  • 周期类型选择“Day”，interval取值同point，可忽略，表示每几天召开一次。
  • 周期类型选择“Week”，interval取值表示隔几周。如填2，则表示每两周的point召开。
  • 周期类型选择“Month”，interval取值表示隔几月。如填2，则表示每两月的point召开。

默认值都为1。

weekDays

WeekDayParam[1..*]

1..n

周期内的会议召开点，可以填多个。

USM内置会议仅支持填入1个元素。
表5 周期性会议召开点参数 WeekDayParam

参数名

类型

约束

描述

weekth

Int

0.1

仅当周期类型选择“Month”时生效，表示月中的第几周。

point

Int[0..*]

1..n

周期内的会议召开点。如果选择“Day”类型，则point参数只支持填入一个元素，多填入的将会被忽略。

例如：

  • 周期类型选择“Day”，point填入2，则表示每两天召开一次。
  • 周期类型选择“Week”，point中填入1和3，则表示每周一和周三召开会议。0表示周日。
  • 周期类型选择“Month”，point中填入12和20，则表示每个月的12号和20号召开会议。USM内置会议仅支持填入1个日期，不支持填入多个 日期。
